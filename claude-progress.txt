- **Fix**: Adicionado auto-repair em `ensureNotificationsTable()` que verifica se colunas existem e se `type` é VARCHAR(50). Se não for, executa ALTER TABLE automaticamente.
- **Arquivo**: `app/Services/NotificationService.php`

#### BUG-2: MonitoringService — Unknown column 'status'
- **Causa raiz**: Tabela `system_monitoring` criada anteriormente sem coluna `status`. O `CREATE TABLE IF NOT EXISTS` não recria tabelas existentes, então a nova coluna nunca foi adicionada.
- **Fix**: Adicionado `SHOW COLUMNS` check + `ALTER TABLE ADD COLUMN status` se coluna não existir.
- **Arquivo**: `app/Services/MonitoringService.php`

#### BUG-3: BaseController — Mensagem de erro exposta em produção
- **Causa raiz**: `withErrorHandling()` concatenava `$e->getMessage()` na resposta JSON para qualquer exceção Throwable, expondo informações internas (paths, queries, nomes de tabelas) em produção.
- **Fix**: Adicionado check de `APP_ENV` — em produção retorna mensagem genérica "Erro interno do servidor".
- **Arquivo**: `app/Controllers/BaseController.php`

#### BUG-4: Database.php — SQL Injection em identifiers
- **Causa raiz**: Métodos `insert()`, `update()`, `delete()`, `select()`, `exists()`, `count()` interpolavam nomes de tabela e coluna diretamente no SQL sem sanitização.
- **Fix**: Adicionado método `sanitizeIdentifier()` com regex whitelist + backtick escaping. Aplicado em todos os 6 métodos CRUD.
- **Arquivo**: `app/Database.php`
